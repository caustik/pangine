# Research tests

This directory contains bounded experiments and characterization oracles. They run in the ordinary test suite so their results remain reproducible, but they do not automatically define Pangine's accepted semantics.

A research test may intentionally capture a known limitation, compare unapproved representations, or use illustrative model constants. Passing means that the experiment still produces its documented result. Promotion into ordinary regression coverage requires a separate semantic decision and removal of the research qualification.

`unified_concept_state.rs` exercises current API behavior around direct root boundaries, structural repetition, finite induction crossover, and generic-match swamping. The corresponding test-only internal prototype lives under `src/engine/research/` because it needs access to matcher internals. Its question-support experiments verify that disjoint source-keyed partitions reduce to the same canonical support Concept as combined state while keeping the proposed numeric evaluator provisional.

The same internal module also contains the bounded oracle that preceded the production experience-set implementation. It constructs real explicit and global/null Observations, then checks exact replay, duplicate recursive observations, unequal same-observer payloads, structural multiplicity, input order, and partition regrouping. Those cases now have ordinary regression coverage because the experience rule was separately accepted and implemented; keeping the oracle here records how the production decision was falsified before promotion.

`classified_collections.rs` tests a smaller, illustrative representation in which relevance, correlation, and observation are three classes of recursive collection. It verifies the identities and reduction laws the classes can share without forcing their entries into one universal tuple. It deliberately keeps a multi-entry Correlation distinct from a relevance composition of singleton Correlations because choosing an identity between those shapes remains an open semantic decision. It also characterizes why the current signed `Relevance` cannot serve as a partition-independent coefficient for the whole family.
